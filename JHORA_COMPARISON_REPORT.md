## Analysis & Recommendations

### 1. Calculation Divergence
- **Astronomical vs. Linear**: This application uses a fully astronomical recursive model where every sub-boundary (MD down to Deha) is a true solar return. JHora appears to use linear interpolation within the Mahadasha cycle. This results in significant drift at the Deha level while maintaining MD alignment.
- **Precision**: The current convergence threshold (1e-9) is excellent for planetary positions but results in sub-minute jitter at the Deha level (~1 minute duration).

### 2. Systematic Time Offsets
- Found offsets of ~53m and ~3h across different charts. This suggests a variance in UT vs Local Time handling or Delta-T corrections for historical dates.

### 3. Recommendations
- **Sub-period Mode**: Implement an optional 'Linear Sub-Dasha' mode to achieve bit-parity with JHora.
- **Delta-T Synchronization**: Align the ephemeris Delta-T model with JHora's settings for better historical accuracy.
